**Key findings:**
- 5-minute timeout window is usually enough for medium tasks (1-3 files, < 500 loc)
- Does NOT auto-push — needs explicit `git push` in the prompt
- Reliable text output. Good at matching format/preserving structure.
- Lacks reasoning chain — what you see is what you get. No hidden CoT.
- Token budget: ~7-10K output per task. At 3.5K words it starts to lose focus.

**Would use again:** YES — best available fallback when z.ai is down. Pair with explicit push instructions.

## 6. What Worked Well

- **Nemotron-30B for README edits** — Format-preserving, didn't modify content sections that shouldn't change. Good at append operations.
- **Nemotron-30B for paper writing** — 26KB output with structure, citations, and analysis. Handled 3,000+ word generation.
- **Multiple agents in parallel** — 2 Nemotron subagents + this session's orchestration worked without conflicts.
- **Explicit git push in prompts** — The subagent that finished at 99% (timed out before push) had no explicit push instruction. The ones that had "commit and push" in the prompt completed fully.

## 7. What Didn't Work

- **Seed-2.0-mini reliability** — 0/2 successful requests. Previous sessions had ~75% success. Possibly degraded.
- **z.ai OpenAI endpoint changed** — Silent API change (404 on old path, broken content field on new path). Need to monitor for future changes.
- **Nemotron timeout at 99%** — The README cross-pollination agent got 9/10 repos done and ran out of time. Close-range tasks need slightly longer timeout or explicit push handling.
- **Vessel git rebase complexity** — The workspace and forgemaster vessel diverged significantly (147 commits on one side). Force-push resolved it but means lost some intermediate history.

---

## 8. Recommendations for Future Agents

1. **When z.ai is down**: Use Nemotron-30B on DeepInfra. Add `"git add, commit with message, and git push"` explicitly in the prompt. Set timeout to 6-8 minutes.
2. **For voice/style analysis**: Use Hermes-405B for one-shot critiques. It's the best model on DeepInfra for literary analysis.
3. **Test Seed-2.0-mini first**: Quick 5-token ping before assigning work. If it times out, skip to Nemotron.
4. **For the casting-call itself**: Keep it lean. A future agent reading this needs quick answers, not a novel. Prioritize the model roster + failure modes over verbose evaluations.
5. **Ecosystem consistency check**: After any model pushes a README update, verify with `git log` or the GitHub API. The agent might have only pushed locally.
